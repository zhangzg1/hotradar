# AI 分析指南

本文件为 Claude 提供热点评分析时的详细参考，确保分析质量与一致性。

## 分析维度

对每条采集结果，需要从以下 6 个维度进行分析：

### 1. 真实性判断 (isReal)

判断内容是否为真实有价值的信息。

**应标记为 false 的情况：**
- 标题党：标题严重夸大，内容无法支撑
- 假新闻：传播未经证实的信息，来源不可靠
- 营销软文：本质是产品推广或广告
- 重复搬运：完全复制其他来源的内容，无新增信息
- 纯情绪发泄：没有实质内容，仅表达情绪

**应标记为 true 的情况：**
- 来自可靠来源的原创内容
- 包含可验证的事实和数据
- 有明确的信息增量

### 2. 相关性评分 (relevance: 0-100)

评估内容与监控关键词的直接关联程度。

| 分数范围 | 含义 | 示例 |
|----------|------|------|
| 0-30 | 完全不相关 | 关键词"Claude"，内容讲的是 Claude 的同名人物 |
| 30-50 | 间接相关 | 关键词"Claude"，内容讨论的是 AI 对话机器人的通用问题，未提及 Claude |
| 50-65 | 有一定关联 | 关键词"Claude"，内容提到了 AI 助手领域，但未直接讨论 Claude |
| 65-80 | 直接相关 | 关键词"Claude"，内容直接讨论了 Claude 的某个功能或更新 |
| 80-100 | 高度相关 | 关键词"Claude"，内容深度分析了 Claude 的技术细节、使用体验等 |

**重要原则：**
- 仅仅属于同一领域但未提及关键词的内容，相关性应低于 40 分
- 内容必须直接讨论、提及关键词才能获得 60 分以上
- 只是间接沾边（如同类产品、同领域但不同主题）应给 30-50 分

### 3. 相关性理由 (relevanceReason)

用一句话解释为什么给出这个相关性分数。

**好的示例：**
- "文章详细介绍了 Claude 的最新功能更新，直接讨论了关键词主题"
- "内容讨论了 AI 对话助手的通用趋势，但未直接提及 Claude"

**不好的示例：**
- "相关"
- "有点关系"

### 4. 关键词提及 (keywordMentioned)

判断内容是否直接提及了监控关键词或其等价表述。

- true：内容中明确出现了关键词的原文、常见缩写、中英文名称等
- false：内容中完全没有出现关键词的任何形式

注意：即使相关性很高，如果关键词未被直接提及，也应设为 false。

### 5. 重要程度 (importance)

| 级别 | 含义 | 判定标准 |
|------|------|----------|
| urgent | 紧急重要 | 突发事件、重大发布、安全漏洞等需要立即关注 |
| high | 高度重要 | 重要更新、深度分析、权威报道 |
| medium | 中等重要 | 常规讨论、使用经验分享、一般性报道 |
| low | 一般 | 简单提及、列表收录、次要信息 |

### 6. 摘要 (summary)

用一句话说明此内容与监控关键词的关联。注意不是介绍内容本身，而是说明"此内容与关键词的关系是什么"。

**格式：** "此内容与【{关键词}】的关联：{具体说明}"

**好的示例：**
- "此内容与【Claude】的关联：报道了 Anthropic 最新发布的 Claude 3.5 模型的性能提升"
- "此内容与【GPT】的关联：分析了 OpenAI 在 GPT-5 上的研发方向和技术路线"

**不好的示例：**
- "这是一篇关于 AI 的文章"
- "内容介绍了新功能"

## 三层过滤规则

分析完成后，对每条结果依次应用以下过滤：

1. **真假过滤**：`isReal == false` → 过滤掉
2. **相关性阈值**：`relevance < 50` → 过滤掉
3. **关键词提及组合**：`keywordMentioned == false && relevance < 65` → 过滤掉

只有通过全部三层过滤的结果才保留在最终输出中。

## 分析效率建议

- 对同一批结果，可以批量分析，但每条结果必须独立判断
- 如果标题和摘要已经足够判断，不需要深入分析全文
- 对于明显不相关的内容（如关键词是技术名词但内容是同名人物），可以快速判定为低相关性
- 关键词预匹配结果可以作为参考，但不应成为唯一依据
